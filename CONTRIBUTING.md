# Руководство по вкладу

Спасибо за интерес к проекту! Улучшение перевода, исправления, новые агенты для российского рынка — любой вклад приветствуется.

## Что можно сделать

### 1. Улучшить перевод

184 апстрим-агента переведены пакетно с помощью Claude Sonnet. Если найдёте нечитаемые места или не свойственные русскому языку обороты — присылайте PR.

При переводе:
- Не переводите буквально — используйте естественный русский технический стиль
- Переводите комментарии внутри блоков кода
- Сохраняйте формат frontmatter (name, description, color, emoji, vibe)
- Устоявшиеся английские термины (TensorFlow, PyTorch, RAG, MLOps, LLM, API, OAuth, REST и т.д.) оставляйте на английском
- Блоки кода, пути к файлам, URL, команды (cat, ls, grep, git, npm) — не трогать

### 2. Создать агентов для российского рынка

PR с агентами под российские платформы и сценарии приветствуются:

- **Платформы РФ**: VK (ВКонтакте), Telegram bot operator, Yandex.Дзен, Rutube
- **E-commerce**: Wildberries seller, Ozon, Яндекс.Маркет, Авито продавец
- **Финтех**: Тинькофф-style PM, СБП интеграция, Сбер API
- **SEO / маркетинг**: Yandex SEO, ВК Реклама, MyTarget, Habr стратегия
- **Compliance**: ФЗ-152, ФЗ-149, КИИ, ФЗ-187
- **Корпоративные**: 1С интеграция, СБИС, российские ВКС

### 3. Улучшить существующий контент

Неточный перевод, устаревший пример кода или лучшая формулировка — правьте смело.

## Формат файла агента

```markdown

name: Имя агента
description: Одна фраза, что делает агент
color: цвет (или #HEX)
emoji: эмодзи
vibe: одна строка — характер агента
---

# Имя агента

Вы — **Имя агента**, [одна фраза-позиционирование].

## Идентичность и память
- **Роль**: конкретная роль
- **Характер**: черты характера
- **Память**: что помните
- **Опыт**: в чём сильны

## Главная миссия
...

## Критические правила
...

## Технические артефакты
Примеры кода, шаблоны, фреймворки

## Рабочий процесс
Пошаговый workflow

## Стиль коммуникации
Примеры тона и фраз

Количественные критерии
```

## Красные линии контента

PR с этим нарушением будут закрыты.

### 1. Без привязки к конкретному работодателю / бренду

Агенты — это «роль и методология», а не идентичность сотрудника. Так нельзя:

```
Вы — Инженер XX в компании XX, мировом лидере в решениях XX...
- Роль: Инженер XX, сотрудник группы XX
```

Правильно (нейтрально):

```
Вы — Инженер XX, опытный практик в XX. В многочисленных проектах XX...
- Роль: Инженер XX — методология и реализация в XX
```

**Исключение**: модели оборудования, названия ПО или протоколов (например, «Bullmer S90 PRO», «Adobe Premiere Pro», «PCI DSS») — техническая ссылка, не реклама.

### 2. Без API / плагинов конкретных инструментов в теле prompt

Prompt агента должен работать в любом LLM, в любом инструменте. Помещение названий API, способов вызова, ссылок на специфический инструмент в теле prompt мусорит пользователям в других контекстах.

### 3. Без скрытого маркетинга под видом «правок документации»

PR с заголовком «docs: правки» но реальное содержимое — внешние ссылки, упоминание собственного продукта, якоря SEO — будут обработаны по реальному содержимому, не по заголовку.

## Отправка PR

1. Сделайте fork
2. Создайте ветку: `git checkout -b add-xxx-agent`
3. Напишите контент, проверьте формат локально
4. Откройте PR с коротким описанием

## Соглашения

- Файлы с LF переносами (не CRLF)
- Один PR — одно изменение (не смешивайте перевод и добавление)
- Commit message на русском или английском
